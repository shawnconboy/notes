File I/O in C++

What is fstream?

fstream is part of the C++ Standard Library and allows you to read from and write to files. It's found in the <fstream> header and gives you control over file input/output (I/O).

Types of File Streams

  • 🔹 ifstream – Input file stream (for reading)
  • 🔹 ofstream – Output file stream (for writing)
  • 🔹 fstream – Can do both reading and writing

Include the Header


#include <fstream>

Writing to a File

This example writes favorite foods to a file called foods.txt:


#include <iostream>
#include <fstream>
using namespace std;

int main() {
    ofstream outFile("foods.txt");

    if (outFile.is_open()) {
        outFile << "Pizza\n";
        outFile << "Sushi\n";
        outFile << "Ice Cream\n";
        outFile.close();
    } else {
        cout << "Failed to open file." << endl;
    }

    return 0;
}

Reading from a File

This reads and prints the contents of foods.txt:


#include <iostream>
#include <fstream>
#include <string>
using namespace std;

int main() {
    ifstream inFile("foods.txt");
    string food;

    if (inFile.is_open()) {
        while (getline(inFile, food)) {
            cout << "I love: " << food << endl;
        }
        inFile.close();
    } else {
        cout << "Unable to open file." << endl;
    }

    return 0;
}

File Opening Modes

  • 📁 ios::in – Open for reading
  • 📁 ios::out – Open for writing
  • 📁 ios::app – Append to the end
  • 📁 ios::trunc – Truncate (erase contents)
  • 📁 ios::binary – Binary mode

Append Mode Example

Add a new food item to the end of the file:


ofstream outFile("foods.txt", ios::app);
outFile << "Tacos\n";

How File Streams Are Like cin and cout

File streams use the same operators as console input/output:

  • 🔹 cout << "Pizza"; – Console output
  • 🔹 outFile << "Pizza"; – File output
  • 🔹 cin >> input; – Console input
  • 🔹 inFile >> input; – File input

So if you know how to use cin and cout, you’re already halfway there with file streams!

Key Tips

  • ✅ Always check if the file opened successfully.
  • ✅ Don’t forget to .close() your files.
  • ✅ Use getline() for reading full lines.
  • ✅ File streams behave like cin/cout but read/write to files instead of the console.
